Click any smell to see the alternative.
Error handling
HIGH
Silent error discard
if err != nil { return } with no logging or wrapping — caller gets zero context
▶
Alternative
Return a wrapped error: return fmt.Errorf("operationName: %w", err). For non-critical paths, log at the point of discard with enough context to reconstruct what happened.
HIGH
Blank-identifier swallow
result, _ := someCall() — error silently discarded
▶
Alternative
Always assign the error: result, err := someCall() then handle it. If you genuinely cannot act on it, at minimum _ = err with a comment explaining why — but that should be extremely rare.
MED
Unwrapped error return
Returning raw err — stack context lost
▶
Alternative
Use fmt.Errorf("context: %w", err) so callers can still use errors.Is / errors.As on the chain. For richer structured errors, define a domain error type and wrap it.
MED
Panic in library code
panic() outside of main/init
▶
Alternative
Return an error value instead. Reserve panic for programmer errors that represent an impossible state (e.g. index out of range in internal logic). If you must panic, document it clearly in the package godoc.
MEDOBSCURE
Sentinel error compared with ==
== io.EOF instead of errors.Is() — fails after wrapping
▶
Alternative
Always use errors.Is(err, io.EOF). It unwraps the chain automatically, so it works regardless of how many layers of %w wrapping sit between your code and the original sentinel.
LOWOBSCURE
errors.As target not a pointer
errors.As(err, myErr) — silently never matches
▶
Alternative
Pass a pointer to the target: var target *MyError; errors.As(err, &target). The function signature accepts any so the compiler won't catch the missing & — a linter rule or unit test is the only reliable guard.
Concurrency
HIGH
Goroutine leak
Goroutine launched with no cancellation path
▶
Alternative
Pass a context.Context into every goroutine and select on ctx.Done() as an exit path. For worker pools, use a sync.WaitGroup and close a done channel to signal shutdown. Tools like goleak can catch leaks in tests.
HIGH
Unlocked shared state
Struct fields mutated across goroutines without mutex or atomic
▶
Alternative
Protect with a sync.Mutex or use sync/atomic for scalar values. For read-heavy workloads, prefer sync.RWMutex. Run tests with -race — the Go race detector catches almost all of these at test time.
MED
Mutex-protected map with value copy
Modifying a struct copy retrieved from a map outside the lock
▶
Alternative
Store pointers in the map (map[K]*V) so you're mutating the actual value. Or do the full read-modify-write inside the lock. Never let a copy escape the critical section.
HIGHOBSCURE
Loop variable capture in goroutine
go func() { use(i) }() without shadowing — all goroutines see final value (pre-Go 1.22)
▶
Alternative
Shadow the variable before the goroutine: i := i; go func() { use(i) }(). Or pass it as a parameter: go func(i int) { use(i) }(i). Go 1.22+ fixes loop variable semantics, but your checker should still flag this for older targets.
MEDOBSCURE
WaitGroup counter goes negative
wg.Done() called more times than wg.Add()
▶
Alternative
Call wg.Add(n) before spawning goroutines, not inside them. Use defer wg.Done() as the first line of the goroutine body to guarantee it fires exactly once even on panic paths.
MEDOBSCURE
Channel send to nil channel
Sending to a var ch chan T (nil) — blocks forever
▶
Alternative
Always initialise channels with make(chan T) before use. If a channel is conditionally used, guard sends with a nil check or use a select with a default case to avoid blocking.
LOWOBSCURE
sync.Mutex copied by value
Passing a struct containing a sync.Mutex by value
▶
Alternative
Always pass structs containing mutexes by pointer. The go vet tool and staticcheck both flag this — ensure your CI runs them.
SQL / Injection
HIGH
String-concatenated query
fmt.Sprintf("SELECT ... WHERE id='%s'", input)
▶
Alternative
Use parameterised placeholders: db.QueryContext(ctx, "SELECT ... WHERE id = ?", id). The driver handles escaping. For complex queries, consider sqlx named params or a query builder like squirrel.
MED
Rows never closed
db.Query() without deferred rows.Close()
▶
Alternative
Add defer rows.Close() immediately after the nil-error check. Also check rows.Err() after the iteration loop — errors during scanning are only reported there.
HIGHOBSCURE
Dynamic ORDER BY / table name
Column/table names can't use parameterised placeholders
▶
Alternative
Validate against a strict allowlist of known column names before interpolating: allowed := map[string]bool{"name": true, "created_at": true}. Reject anything not in the map. Never trust user input directly for identifiers.
Resource management
HIGH
Deferred close inside loop
defer f.Close() inside a loop — queues until function exit
▶
Alternative
Extract the loop body into a helper function and defer inside that: func process(path string) error { f, _ := os.Open(path); defer f.Close(); ... }. Or close explicitly at the end of each iteration without defer.
MED
Context ignored
Function accepts ctx but never passes it to I/O calls
▶
Alternative
Pass ctx to every I/O call that accepts it: db.QueryContext(ctx, ...), http.NewRequestWithContext(ctx, ...). If a dependency doesn't accept context, wrap it with a goroutine and a select on ctx.Done().
MEDOBSCURE
http.Response body not drained
Closing body without reading it — prevents connection reuse
▶
Alternative
Before closing, drain the body: io.Copy(io.Discard, resp.Body). This allows the underlying TCP connection to be returned to the pool. Pair with defer resp.Body.Close() as usual.
LOWOBSCURE
time.After in hot loop
Allocates a new timer channel each call — leaks until fired
▶
Alternative
Use t := time.NewTimer(d) and defer t.Stop(), then reuse with t.Reset(d) on each iteration. This avoids the allocation overhead and the goroutine leak associated with abandoned time.After channels.
Type / interface
MED
Interface holding nil pointer
Typed nil assigned to interface — if x == nil check silently fails
▶
Alternative
Return a bare nil (untyped) when the interface should be nil: var err error = nil; return err is fine, but var p *MyError = nil; return p as an error is not. Use reflect.ValueOf(v).IsNil() if you must check dynamically.
LOW
Over-broad any parameter
Accepting any instead of a narrow interface
▶
Alternative
Define the smallest interface that satisfies your needs: type Namer interface { Name() string }. If multiple types are legitimately needed, use a union via generics: func F[T int | string](v T).
MEDOBSCURE
Slice header escape
Returning a sub-slice keeps the full backing array alive
▶
Alternative
Copy the sub-slice into a fresh allocation: result := make([]T, len(sub)); copy(result, sub). The original large backing array can then be GC'd.
LOWOBSCURE
append result discarded
append(slice, val) without reassigning
▶
Alternative
Always reassign: slice = append(slice, val). This is one of the easiest AST-level checks to implement — any append call whose return value is not assigned is the smell.
Performance
LOWOBSCURE
strings.Builder copied by value
Copying a strings.Builder after first write
▶
Alternative
Always pass *strings.Builder by pointer. The type has a copyCheck mechanism that panics on copy after first use — but only at runtime, not compile time. Pass by pointer from the start.
LOWOBSCURE
Large struct in map value
Map values not addressable — every write copies the full struct
▶
Alternative
Use map[K]*V so only the pointer is stored and copied. The threshold is subjective, but any struct over ~64 bytes or with heap-allocated fields (slices, maps, strings) is a candidate.
Exception handling
HIGH
Bare except
except: catches everything including SystemExit, KeyboardInterrupt
▶
Alternative
Catch the most specific exception type you can handle: except ValueError as e:. If you need a broad catch-all, use except Exception as e: which excludes system-exiting exceptions, and always log or re-raise.
HIGH
Silent swallow
except ...: pass — error consumed with no log or re-raise
▶
Alternative
At minimum: except SomeError: logger.warning("...", exc_info=True). If suppression is intentional, add a comment. contextlib.suppress(ErrorType) is an explicit, readable way to intentionally swallow a known exception type.
MED
Raising generic Exception
raise Exception("...") — callers can't catch by type
▶
Alternative
Define domain-specific exception classes: class ValidationError(ValueError): .... Callers can then write except ValidationError without catching unrelated errors.
MEDOBSCURE
Exception chaining lost
raise NewError() from None discards the original traceback
▶
Alternative
Use implicit chaining (raise NewError() inside an except block) or explicit chaining (raise NewError() from original_exc) to preserve the full traceback. Only use from None when you're deliberately hiding implementation details at an API boundary.
LOWOBSCURE
Re-raising wrong exception
except ValueError: raise TypeError() without from e
▶
Alternative
Preserve the original: except ValueError as e: raise TypeError("...") from e. The from e sets __cause__ so the full chain appears in tracebacks.
SQL / Injection
HIGH
f-string / %-format query
cursor.execute(f"SELECT ... WHERE id='{uid}'")
▶
Alternative
Use parameterised queries: cursor.execute("SELECT ... WHERE id = %s", (uid,)). The DB-API driver handles escaping. For ORMs like SQLAlchemy, use bound parameters or the ORM query API — never text() with string formatting.
HIGH
Shell injection via subprocess
subprocess.run(cmd, shell=True) with user input
▶
Alternative
Pass a list instead of a string and drop shell=True: subprocess.run(["git", "clone", user_url]). The OS handles argument separation safely. If you need shell features, use shlex.quote() on each user-supplied token.
HIGH
eval / exec on external input
eval(user_input) — arbitrary code execution
▶
Alternative
For config/data: use json.loads(), ast.literal_eval() (safe subset of Python literals only), or a schema validator like Pydantic. For dynamic dispatch: use a dispatch table (dict of callables) keyed by string.
HIGHOBSCURE
pickle.loads on untrusted data
Deserialising pickle bytes executes embedded bytecode — RCE
▶
Alternative
Use a format that can't execute code: JSON, MessagePack, or Protocol Buffers. If pickle is required for internal caching, sign the payload with hmac and verify before loading. Never load pickle from a network or user-supplied source.
HIGHOBSCURE
yaml.load without Loader
yaml.load(data) — can instantiate arbitrary Python objects
▶
Alternative
Always pass a safe loader: yaml.safe_load(data) or yaml.load(data, Loader=yaml.SafeLoader). The safe loader only handles standard YAML types and cannot instantiate Python objects.
MEDOBSCURE
Dynamic ORDER BY / table name
Column/table names can't use parameterised placeholders
▶
Alternative
Validate against an allowlist: ALLOWED_COLS = {"name", "created_at"}; assert col in ALLOWED_COLS. Use psycopg2.sql.Identifier for safe identifier quoting in PostgreSQL contexts.
Mutability traps
HIGH
Mutable default argument
def f(items=[]) — list shared across all calls
▶
Alternative
Use None as the sentinel and create the default inside the function: def f(items=None): items = items if items is not None else []. This is idiomatic Python and avoids the shared-state issue entirely.
MED
Late-binding closure
Lambda in a loop captures loop variable by reference
▶
Alternative
Capture by value using a default argument: lambda x=x: use(x). Or use a factory function: def make_handler(x): return lambda: use(x). Both force evaluation at definition time.
MEDOBSCURE
Class-level mutable attribute
class Foo: items = [] — shared across all instances
▶
Alternative
Move mutable state into __init__: def __init__(self): self.items = []. Class-level attributes are fine for immutable constants and class-wide configuration, not for per-instance mutable state.
LOWOBSCURE
Aliased list/dict not copied
b = a — mutations through b affect a
▶
Alternative
Shallow copy: b = a.copy() or b = list(a). Deep copy when nested structures are involved: import copy; b = copy.deepcopy(a). For read-only access, consider typing the parameter as Sequence to make intent explicit.
Type / safety
MED
Implicit None return used as value
Function sometimes returns a value, sometimes falls off the end
▶
Alternative
Be explicit about the return type. If the function can return nothing, annotate it as Optional[T] (or T | None) and add a deliberate return None at every exit point. Mypy with --strict will flag inconsistent return paths.
LOW
Missing type annotations on public API
Public functions without hints — no static analysis
▶
Alternative
Add annotations incrementally. Start with public-facing functions, then work inward. Enable mypy or pyright in CI. Even partial annotations catch a significant class of bugs without requiring a full rewrite.
MEDOBSCURE
isinstance with list literal
isinstance(x, [str, bytes]) — raises TypeError at runtime
▶
Alternative
Use a tuple: isinstance(x, (str, bytes)). The second argument must be a type or a tuple of types — lists are not accepted and will raise TypeError at the call site.
LOWOBSCURE
Comparison to True/False with ==
if x == True — coercion-prone
▶
Alternative
Use if x: for truthiness checks or if x is True: for strict boolean identity. PEP 8 explicitly discourages == True/== False comparisons. Flake8 rule E712 flags this.
Resource management
MED
File / connection not context-managed
Opening files or DB connections outside a with block
▶
Alternative
Always use with open(...) as f: and with db.connect() as conn:. If a class doesn't implement the context manager protocol, wrap it with contextlib.closing().
MEDOBSCURE
Generator not fully consumed
Generator wrapping a resource abandoned mid-iteration
▶
Alternative
Use contextlib.contextmanager with try/finally to ensure cleanup runs even if the caller breaks early. Or use with-style generators via @contextlib.contextmanager so the resource is tied to a context manager, not the generator's lifetime.
LOWOBSCURE
threading.Lock not released on exception
Manual acquire/release without a with block
▶
Alternative
Use the lock as a context manager: with lock: .... This guarantees release even if an exception is raised inside the block.
Import / module
MEDOBSCURE
Circular import
Module A imports B which imports A — partial initialisation
▶
Alternative
Break the cycle by moving shared types into a third module that neither A nor B imports. For unavoidable cases, use a local import inside the function body to defer resolution until call time.
LOWOBSCURE
Star import in non-init code
from module import * — namespace pollution
▶
Alternative
Import only what you use: from module import ClassA, func_b. Star imports are acceptable in __init__.py for re-exporting a public API, but should be controlled with an explicit __all__ list in the source module.
LOWOBSCURE
Module-level side effects
I/O or heavy computation at import time
▶
Alternative
Move side-effectful logic into functions or a main() entrypoint guarded by if __name__ == "__main__":. For expensive initialisation, use lazy loading patterns or module-level __getattr__.
Async
HIGH
Blocking call inside async function
time.sleep() or requests.get() in async def
▶
Alternative
Use async equivalents: await asyncio.sleep(), httpx.AsyncClient, aiofiles. For blocking work that can't be replaced, offload with await asyncio.get_event_loop().run_in_executor(None, blocking_fn).
MEDOBSCURE
Unawaited coroutine
Calling async def without await — returns coroutine object, does nothing
▶
Alternative
Always await coroutines or schedule them with asyncio.create_task(). Python 3.11+ emits a RuntimeWarning for unawaited coroutines. Enable warnings-as-errors in tests to catch these early.
MEDOBSCURE
asyncio.get_event_loop() in production
Deprecated — creates a new loop if none running
▶
Alternative
Use asyncio.get_running_loop() inside async code (raises if no loop is running, which is the correct behaviour). For entrypoints, use asyncio.run(main()) which manages the loop lifecycle cleanly.
Type system abuse
HIGH
Unguarded non-null assertion
obj!.property when obj can genuinely be null
▶
Alternative
Use optional chaining: obj?.property. If null is truly impossible at that point, add a guard that throws a meaningful error: if (!obj) throw new Error("obj must exist"). Reserve ! for cases where you've verified non-null just above.
HIGH
any escape hatch
as any to silence type errors
▶
Alternative
Use unknown instead of any for genuinely unknown input — it forces you to narrow the type before use. For library interop, use precise overrides or augment the type definition rather than casting to any.
MED
Unsafe type assertion
data as SomeType without runtime validation
▶
Alternative
Validate at the boundary with a schema library: zod, valibot, or io-ts. These parse and validate simultaneously, giving you a typed value only if the shape actually matches.
MEDOBSCURE
Enum used as bitwise flags
Numeric enums combined with | — TS doesn't model this
▶
Alternative
Use a const object with explicit bit values and a union type: const Flags = { Read: 1, Write: 2 } as const; type Flag = typeof Flags[keyof typeof Flags]. Or use a Set<FlagEnum> which is more idiomatic and type-safe in TS.
MEDOBSCURE
Widening via object literal spread
{ ...typedValue, extra: 1 } disables excess property checking
▶
Alternative
Explicitly type the result: const result: MyType = { ...typedValue, extra: 1 } — TS will then validate the shape. Or define an extended type: type Extended = MyType & { extra: number }.
MEDOBSCURE
Overlapping overload signatures
Overloads that overlap — wrong overload selected silently
▶
Alternative
Order overloads from most specific to least specific. Use discriminated unions instead of overloads where possible — they're easier to narrow and don't have ordering issues.
LOWOBSCURE
Mutable tuple typed as array
[string, number] widened to Array<string | number>
▶
Alternative
Use as const to preserve tuple structure: const pair = ["a", 1] as const. Or annotate the type explicitly: const pair: [string, number] = ["a", 1]. This preserves positional types and prevents wrong-order access.
LOWOBSCURE
Namespace merging side-effect
Unintentional global namespace augmentation
▶
Alternative
Isolate augmentations in a dedicated .d.ts file and import it only where needed. Add export {} at the top of any file doing declare global to ensure it's treated as a module, not a script.
Async / promises
HIGH
Floating promise
Async call without await or .catch()
▶
Alternative
Always await or chain .catch(). If fire-and-forget is intentional, be explicit: void someAsync(). ESLint rule @typescript-eslint/no-floating-promises catches these automatically.
HIGH
Empty catch block
catch { } — exception silently swallowed
▶
Alternative
At minimum log: catch (e) { console.error("context", e); }. For expected errors, narrow the type and handle specifically. Use a top-level error boundary or process.on('uncaughtException') as a last-resort safety net, not a replacement.
MED
Sequential awaits in a loop
for (const x of list) { await fetch(x) } — runs serially
▶
Alternative
Use await Promise.all(list.map(x => fetch(x))) for full parallelism. For concurrency-limited batching, use a library like p-limit or chunk manually with Promise.all over slices.
HIGHOBSCURE
async function in Array.forEach
items.forEach(async item => { await ... }) — promises ignored
▶
Alternative
Replace with await Promise.all(items.map(async item => { ... })). The map collects the promises; Promise.all awaits them. Alternatively use a for...of loop with await if serial execution is needed.
MEDOBSCURE
Promise constructor anti-pattern
new Promise(async (res, rej) => { ... }) — thrown errors not caught
▶
Alternative
Use an async function directly and await it. Only use new Promise() to wrap callback-based APIs (promisify pattern), and never with an async executor. If you must, wrap the entire body in a try/catch and call rej(e) in the catch.
MEDOBSCURE
Unhandled rejection in Promise.all
First rejection kills all — others silently abandoned
▶
Alternative
Use Promise.allSettled() to collect all results regardless of rejection — then inspect each result's status. Or wrap individual promises with .catch() before passing them to Promise.all if partial failure is acceptable.
LOWOBSCURE
Mixed .then() and async/await
Inconsistent error propagation
▶
Alternative
Pick one style per function and stay consistent. Prefer async/await for readability. Reserve .then() chaining for cases where you're deliberately building a pipeline and don't need the surrounding function to be async.
SQL / Injection
HIGH
Template-literal query
db.query(`SELECT ... WHERE id='${id}'`)
▶
Alternative
Use driver parameterisation: db.query("SELECT ... WHERE id = $1", [id]) for pg, or db.execute(sql`SELECT ... WHERE id = ${id}`) with tagged template literal libraries like slonik or postgres.js that handle escaping internally.
HIGH
innerHTML with user input
element.innerHTML = userInput — XSS
▶
Alternative
For plain text: use element.textContent = userInput. For rich content you must render as HTML: sanitise first with DOMPurify.sanitize(userInput) before assigning to innerHTML.
HIGHOBSCURE
Dynamic ORDER BY in ORM
Column names passed to ORM sort methods without allowlist
▶
Alternative
Validate against a typed allowlist: const SORTABLE = ["name", "createdAt"] as const; type SortCol = typeof SORTABLE[number]. Check membership before passing to the ORM. The type system helps but doesn't replace the runtime check for externally-sourced values.
HIGHOBSCURE
Prototype pollution via merge
Deep-merging user JSON can set __proto__ — poisons Object prototype
▶
Alternative
Use a merge library with prototype pollution protection (lodash.merge has had fixes; prefer deepmerge with the isMergeableObject option). Or use JSON.parse(JSON.stringify(obj)) as a safe but blunt deep-clone. Validate input with a schema before merging.
MEDOBSCURE
eval / Function constructor
new Function(userInput)() — arbitrary code execution
▶
Alternative
Use a dispatch table for dynamic behaviour: const handlers: Record<string, () => void> = { greet: () => ... }. For safe expression evaluation, use a sandboxed math library like mathjs rather than eval.
Equality / coercion
MED
Loose equality
== instead of ===
▶
Alternative
Always use === and !==. The only defensible use of == is x == null as a shorthand for x === null || x === undefined — and even then, be explicit if the code is read by a team.
MED
typeof on complex types
typeof obj === "object" also matches null and arrays
▶
Alternative
Write a proper type guard: function isMyType(v: unknown): v is MyType { return v !== null && typeof v === "object" && "field" in v }. This narrows correctly and is self-documenting.
MEDOBSCURE
NaN comparison
x === NaN always false
▶
Alternative
Use Number.isNaN(x) for strict NaN checks (doesn't coerce). Avoid the global isNaN() which coerces strings first: isNaN("hello") returns true, Number.isNaN("hello") returns false.
LOWOBSCURE
Object spread overwrite
{ ...defaults, ...userInput } with unvalidated input
▶
Alternative
Validate userInput with a schema (zod, valibot) before merging, or pick only the keys you expect: const safe = { ...defaults, name: userInput.name }. Explicit is safer than spreading an unknown shape.
React / DOM
HIGH
dangerouslySetInnerHTML with user data
Unsanitised content — XSS
▶
Alternative
Sanitise with DOMPurify: dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(content) }}. Better yet, use a Markdown renderer like react-markdown that never uses innerHTML directly.
MED
Missing dependency in useEffect
Stale closure — effect captures old variable values
▶
Alternative
Include all values read inside the effect in the dependency array. Use the eslint-plugin-react-hooks exhaustive-deps rule — it catches these automatically. For values that change on every render but shouldn't retrigger the effect, use useRef.
HIGHOBSCURE
State mutation without setState
Directly mutating state — React doesn't detect the change
▶
Alternative
Always return a new reference: setState(prev => [...prev, newItem]) for arrays, setState(prev => ({ ...prev, key: val })) for objects. Consider immer via useImmer for complex nested state — it lets you write mutating syntax while producing immutable updates.
MEDOBSCURE
useEffect with async function directly
useEffect(async () => ...) — cleanup never runs
▶
Alternative
Define the async function inside the effect and call it immediately: useEffect(() => { async function load() { ... } load(); return () => { /* cleanup */ }; }, [deps]). The outer function remains synchronous and can return a cleanup.
MEDOBSCURE
Unstable object/function as prop
Inline {} or () => breaks memoised child
▶
Alternative
Stabilise with useMemo for objects: const opts = useMemo(() => ({ x: 1 }), []) and useCallback for functions: const handler = useCallback(() => ..., [deps]). Only needed when the child is wrapped in React.memo.
LOWOBSCURE
Index as React key
key={index} in a sorted/filtered list
▶
Alternative
Use a stable unique identifier from the data: key={item.id}. Index-as-key is only safe for purely static lists that never reorder, filter, or change length. If those conditions aren't guaranteed, a bad key will cause subtle input and animation bugs.
Runtime / JS gotchas
MED
parseInt without radix
parseInt("09") without explicit base
▶
Alternative
Always pass the radix: parseInt("09", 10). For numeric strings without fractions, Number("09") is often cleaner and doesn't have the radix issue. +str coercion also works but is less readable.
HIGHOBSCURE
Floating-point currency arithmetic
0.1 + 0.2 !== 0.3 in financial logic
▶
Alternative
Store and operate on integer cents/pence: const total = priceInCents + taxInCents. For complex financial calculations, use a decimal library like decimal.js or dinero.js. Never use native floats for money.
MEDOBSCURE
delete on object property
delete obj.key de-optimises V8 hidden class
▶
Alternative
Reconstruct without the key: const { key, ...rest } = obj. Or set to undefined if the property must remain in the shape. Both preserve the object's hidden class and avoid the V8 de-optimisation.
MEDOBSCURE
for...in on an array
Iterates enumerable string keys including prototype additions
▶
Alternative
Use for...of for values, arr.forEach(), or a plain indexed loop. Reserve for...in strictly for plain objects where you want to iterate own enumerable keys — even then, guard with hasOwnProperty.
MEDOBSCURE
setTimeout reference lost
Timer started without storing handle — can't be cleared
▶
Alternative
Store the return value: const timerId = setTimeout(...) and clear on cleanup: clearTimeout(timerId). In React, store in a useRef and clear in the useEffect cleanup function.
LOWOBSCURE
typeof check before optional chaining
typeof obj !== 'undefined' && obj.x — redundant
▶
Alternative
Use optional chaining: obj?.x. It handles both null and undefined and is shorter. The typeof guard is only needed in environments where accessing an undeclared variable would throw — which is rare in module-based TS.
Node.js specific
HIGH
Unhandled promise rejection (Node)
No process.on('unhandledRejection') handler — Node 15+ crashes
▶
Alternative
Add a global handler at startup: process.on('unhandledRejection', (reason) => { logger.error(reason); process.exit(1); }). This is a last resort — the real fix is ensuring every promise has a .catch() or try/await at the call site.
HIGHOBSCURE
Path traversal via user input
fs.readFile(userInput) — reads arbitrary files
▶
Alternative
Resolve and validate the path: const resolved = path.resolve(BASE_DIR, userInput); if (!resolved.startsWith(BASE_DIR)) throw new Error("Forbidden"). Never concatenate user input into paths without this check.
MEDOBSCURE
Synchronous fs in async server
fs.readFileSync() in a request handler — blocks event loop
▶
Alternative
Use await fs.promises.readFile() or the fs/promises module. If startup-time synchronous reads are acceptable (loading config once), confine them to the initialisation path before the server starts accepting requests.
MEDOBSCURE
require() inside a hot path
Dynamic require() in frequently-called code — slow on first hit
▶
Alternative
Move require() / import to the top of the module. For genuinely lazy-loaded features, use dynamic import() once at a boundary (route handler init, feature flag check) rather than inside a per-request function.
